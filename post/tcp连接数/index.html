tcp 连接数计算

时间： 2022-06-07 分类： 计算机基础   字数： 522 字 阅读： 2分钟 阅读次数：

今天在知乎上刷到一个很有意思的问题：
linux 的 TCP 连接数量最大不能超过 65535 个，那服务器是如何应对百万千万的并发的？

想起以前学 swoole 时，韩天峰也针对此问题作出过回答。
记录如下,答案来自 知乎用户 :
TCP 协议，能够发出的对外的连接，受限于本机端口数量，每个本机 IP 地址最多能向外发出 65535 个连接。而本机作为服务器时，只监听一个端口，接受外部的连接，是不受本地端口数量限制的。

TCP 描述一个连接时，地址信息包含 4 个元素 (local_ip, local_port, remote_ip, remote_port) 。

先限制本机只有一个IP地址，基 local_ip 是固定的。当向外发出连接时，对方服务器的地址是固定的，即 remote_ip 和 remote_port 是固定的，那么整个 pair 里能够变化的就只有 local_port 了。而local_port 是 16bit，除去 0 有特殊含义外，可用的就是 65535 个端口。所以当本机作为客户端，向外发出连接时，最多只能有 65535 个连接。

但当本机作为服务器时，local_port是固定的，而 remote_ip 和 remote_port 则可以是任意的值。所以此时可以接受的最大连接数就是 2^32 * 2^16=2^48=281,474,976,710,656(即 remote_ip的种类*端口数)，计算中并未去除特殊地址和特殊端口，但量级是没毛病的。

所以，问题的核心就是思考清楚，一个TCP连接的地址构成，知道哪些是可变的，就能估算出来最大的数字了。
