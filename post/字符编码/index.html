字符编码

问题

  1. 为什么需要字符集？
  2. 为什么会出现乱码？
  3. ascii，unicode 和 utf8 之间是一种什么关系？

字符集

在计算机屏幕上看到的是实体化的文字，而在计算机存储介质中存放的实际是二进制的比特(byte)流。
简单来说字符集就是：规定了某个文字对应的二进制数值存放方式（编码）和某串二进制数值代表了哪个文字（解码）的转换关系。

常见字符集：ASCII 字符集、GB2312 字符集、BIG5 字符集、 GB18030 字符集、Unicode 字符集等。

ASCII

1967 年，美国制定了一套字符编码，将字符与二进制位之间的关系，做了统一规定。这被称为 ASCII 。

每个二进制位只有 0 和 1 两种状态，8 个二进制位可以组合成 2 的 8 次方，从 00000000 到 11111111，256 种状态。
而 ASCII 码只有 128 个，一个 字节(byte) 等于 8 个 二进制位(bit)，ASCII 码以二进制的形式存储，只需要一个字节就好了。

Unicode

ASCII 码只有区区 128 个，而世界上的有很多字符，比如说汉字，多达 10万 个，常用的就有 6000 个。很明显，只用 ASCII 码没办法表示。

这时，人们提出了一个新的编码，将世界上 所有 的符号都纳入其中，每一个符号都给予一个独一无二的编码，这就是 Unicode 。

Unicode 的存储问题

Unicode 只是一个符号集，它只规定了符号的二进制代码，却没有规定这个二进制代码应该如何存储。
假设和 ASCII 码一样，单纯的以二进制形式存储，会出现 3 个问题：

  1. 怎么区别 ASCII 还是 Unicode
  2. 英文字符只用一个字节就够了，但是汉字可能要 2-3个字节，如果 Unicode 统一规定，每个符号用三个或四个字节表示，那么每个英文字母前都必然有二到三个字节是 0，这对于存储来说是极大的浪费
  3. 计算机怎么知道三个字节表示一个符号，而不是分别表示三个符号呢

UTF-8

UTF-8 是互联网上使用最广的一种 Unicode 的 实现方式。它是一种 变长的编码方式。它可以使用1~4个字节表示一个符号，根据不同的符号而变化字节长度。

UTF-8 编码规则

  • 对于单字节的符号，字节的第一位设为0，后面7位为这个符号的 Unicode 码。因此对于英语字母，UTF-8 编码和 ASCII 码是相同的。
  • 对于n字节的符号（n > 1），第一个字节的前 n 位都设为 1，第 n + 1 位设为 0，后面字节的前两位一律设为 10。剩下的没有提及的二进制位，全部为这个符号的 Unicode 码。

解答开篇

  1. 为什么会出现乱码？

字符最终会以二进制的方式存储在磁盘中。字符在编码 A 的二进制值和在编码 B 的二进制可能不一样，通过字符集映射后，最终找到的可能是另外一个字符。

  1. ascii，unicode 和 utf-8 之间是一种什么关系？

ascii 和 unicode 应该是一种并列的关系，属于 字符集，即二进制值到字符的映射。
而 utf-8，是 unicode 的一种实现，是基于 unicode 这个字符集，额外制定的一套存储规则，是 编码规则。
